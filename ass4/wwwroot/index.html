Self-evaluation report

Appendix A (A-Level guidelines)

Guidlines Feature Complies? Explanation
1.1 Text Alternatives:
Provide text alternatives for any non-text content so that it can be changed into other forms people need, such as large print, braille, speech, symbols or simpler language.
1.1.1 Non-text Content: We stipulated each image with a corresponding alt explanation to make our website more compatible with people with disabilities.
1.3 Adaptable:
Create content that can be presented in different ways (for example simpler layout) without losing information or structure.
1.3.1 Info and Relationships Our website uses headings and correct design which displays information in a comprehensible way.
1.3.2 Meaningful Sequence Our website displays content in a correct reading sequence. Although some parts of the site are responsive, the meaning of the text stays consistent.
1.3.3 Sensory Characteristics We implemented the sensory characteristics principle by making the components of our website easily understandable without relying on shape, size, visual location, orientation, or sound.
1.4 Distinguishable:
Make it easier for users to see and hear content including separating foreground from background.
1.4.1 Use of Color Our website displays colors in different areas due to aesthetics and not because we try to make our visitors click on specific links.
1.4.2 Audio Control Not applicable
2.1 Keyboard Accessible:
Make all functionality available from a keyboard.
2.1.1 Keyboard Our website is designed to be navigated using the keyboard. By pressing tab, shift-tab and enter, a visitor can easily navigate through the different sections and fill in forms. We use the focus element to highlight the current selected element.
2.1.2 No Keyboard Trap Keyboard focus can be moved to a component and out of it.
2.4 Navigable:
Provide ways to help users navigate, find content, and determine where they are.
2.4.1 Bypass Blocks A link included in the header to skip header and aside focus could be implemented if we did not have css based sections/pages. But right now, the link wouldn't know which section is active. A JS implementation could be used.
2.4.2 Page Titled Our website include a title. In addition, each section is titled as well.
2.4.3 Focus Order A visitor’s focus is directed to the correct (next) section by id.
2.4.4 Link Purpose (In Context) The purpose of each link can be determined from the link text alone.

Browser compatibility report

Methods used:

We examined used HTML and CSS markups using the caniuse.com website. This website compares the compatibility of elements (or attributes) between the most popular browsers. To see a view results, click on the following links to view the compatibility result of their respective element.

table, font-variant, font-weight, widow.

Visual comparison:

By looking at the two leading broswers Chrome and Firefox, we concluded that the main diffrence is the font-weight which although is eqaul to 700 in both browsers, is seems to be bolder in Firefox.

Print style:

Both browsers are behaving as programmed in out print.css file. We carfully chose the order of the elements to fit aesthetically into different pages.

Screen style:

Initially, we wanted to use the CSS attribute flex or CSS grid. However, both attributes are supported in only 65% of the browsers. Therefore, we used the calc() CSS function, which has a 96% support across the browsers. We believe that every person, doesn't matter which browser they are using should have the ability of using our store.

The responsive breaking points this site is using are: max-width: 860px and max-width: 500px. These are handled the same way on each browser and device as we are using the responsive meta tag.
